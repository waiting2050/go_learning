## Redis 通用命令

通用指令是部分数据类型都可以使用的命令，常见的有：

- `KEYS`：查看符合模板的所有 key  
  - ⚠️ 不建议在生产环境使用，可能造成性能问题

- `DEL`：删除一个指定的 key  

- `EXISTS`：判断 key 是否存在  

- `EXPIRE`：给一个 key 设置有效期，到期后该 key 会被自动删除  

- `TTL`：查看一个 key 的剩余有效期  

## Redis String 类型常见命令

### 基本操作
- `SET`：添加或者修改已经存在的一个 String 类型的键值对  
- `GET`：根据 key 获取 String 类型的 value  

### 批量操作
- `MSET`：批量添加多个 String 类型的键值对  
- `MGET`：根据多个 key 获取多个 String 类型的 value  

### 数值操作
- `INCR`：让一个整型的 key 自增 1  
- `INCRBY`：让一个整型的 key 自增并指定步长  
  - 示例：`INCRBY num 2` 让 num 的值自增 2  
- `INCRBYFLOAT`：让一个浮点类型的数字自增并指定步长  

### 条件与过期设置
- `SETNX`：添加一个 String 类型的键值对，前提是这个 key 不存在，否则不执行  
- `SETEX`：添加一个 String 类型的键值对，并且指定有效期  

## Redis Hash 类型常见命令

### 基本操作
- `HSET key field value`：添加或修改 hash 类型 key 的某个 field 的值  
- `HGET key field`：获取 hash 类型 key 的某个 field 的值  

### 批量操作
- `HMSET`：批量添加多个 hash 类型 key 的 field 的值  
- `HMGET`：批量获取多个 hash 类型 key 的 field 的值  

### 全量获取
- `HGETALL`：获取一个 hash 类型 key 中所有的 field 和 value  
- `HKEYS`：获取一个 hash 类型 key 中所有的 field  
- `HVALS`：获取一个 hash 类型 key 中所有的 value  

### 数值操作
- `HINCRBY`：让 hash 类型 key 的某个字段值自增并指定步长  

### 条件添加
- `HSETNX`：添加一个 hash 类型 key 的 field 值，前提是该 field 不存在，否则不执行  

## Redis List 类型常见命令

### 插入元素
- `LPUSH key element ...`：向列表左侧插入一个或多个元素  
- `RPUSH key element ...`：向列表右侧插入一个或多个元素  

### 弹出元素
- `LPOP key`：移除并返回列表左侧的第一个元素（没有则返回 `nil`）  
- `RPOP key`：移除并返回列表右侧的第一个元素  

### 范围查询
- `LRANGE key start end`：返回指定索引范围内的所有元素  

### 阻塞弹出
- `BLPOP key [key ...] timeout`：阻塞式左弹出，等待指定时间直到有元素  
- `BRPOP key [key ...] timeout`：阻塞式右弹出，等待指定时间直到有元素
