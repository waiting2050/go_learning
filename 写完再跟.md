### 核心需求复述
你希望AI扮演「编程导师」的角色，手把手引导你**一步步手动敲完**复刻TikTok的青训项目代码（而非直接照搬AI生成的完整代码），想明确该用Trae的IDE模式还是Solo模式，以及是否需要单独设置智能体来实现这个“分步教学”的需求。

## 一、核心结论：IDE模式 + 专属「编程导师」智能体（最优解）
### 1. 模式选择：优先用「IDE模式」（放弃Solo模式）
| 模式       | 适配性 | 核心原因                                                                 |
|------------|--------|--------------------------------------------------------------------------|
| IDE模式    | ✅ 强适配 | 支持**代码实时交互、分步讲解、断点式教学**，能直接关联你的项目目录，看你写的代码并逐行纠错，完美贴合“一步步敲代码”的需求 |
| Solo模式   | ❌ 不适配 | 偏向“全量代码生成/整体问题解答”，无分步引导、实时代码校验的能力，适合快速出代码但不适合教学 |

### 2. 智能体设置：新建「TikTok编程导师」专属智能体（不复用Solo Coder）
不用复用之前的Solo Coder（它偏向“写代码”而非“教代码”），新建一个聚焦“教学引导”的智能体，优势是：
- 角色更纯粹：全程以“老师”身份沟通，不会混杂“写代码”的思维，引导逻辑更聚焦；
- 配置更精简：只挂载教学必需的MCP，避免冗余功能干扰；
- 记忆更连贯：针对你的TikTok项目形成专属的教学进度记忆，不会和其他任务混淆。

## 二、具体操作步骤（手把手落地）
### 步骤1：新建「TikTok编程导师」智能体并配置
1. 打开Trae → 智能体列表 → 新建智能体 → 命名为「TikTok编程导师」；
2. 挂载**仅需的3个MCP**（足够支撑教学，多了反而乱）：
   - `FileSystem`：读取你写的代码文件、保存教学步骤文档（如“第1步-环境搭建.md”）；
   - `SequentialThinking`：按“从易到难”拆解教学步骤，避免跳步；
   - `MySQL`（可选）：如果你敲数据库相关代码时，导师能帮你验证SQL是否正确；
3. 切换到「IDE模式」→ 关联你的TikTok项目本地目录（关键！让导师能看到你写的代码）。

### 步骤2：给导师设定“教学规则”（复制粘贴到对话框，固化角色）
```
你现在是我的TikTok青训项目专属编程导师，核心职责是引导我一步步手动敲完代码，而非直接给完整代码。请严格遵守：
1. 教学节奏：每次只教1个小步骤（比如“搭建Go后端基础框架”“写用户表建表SQL”），等我完成并反馈后，再教下一步；
2. 讲解方式：先讲“这一步要做什么、为什么这么做”，再给**核心代码片段示例**（不超过10行），最后让我自己写；
3. 纠错逻辑：我写完后，你先指出问题（比如语法错误、逻辑漏洞），再给出修改思路，而非直接改好；
4. 技术栈：全程基于Go（后端）+ Vue3（前端）+ MySQL（数据库），贴合字节青训项目要求；
5. 进度管理：每完成1步，总结当前进度，并告知下一步要学的内容。
```

### 步骤3：设计“从0到1”的教学流程（直接发给导师，启动教学）
把下面的流程发给导师，它会按这个节奏引导你，你只需跟着敲代码即可：
```
请按以下进度引导我完成TikTok复刻项目，每次只推进1步：
第1步：环境搭建（教我安装Go1.22、Vue3、MySQL，验证环境是否正常）；
第2步：项目结构设计（教我创建后端/前端的基础目录结构，解释每个目录的作用，让我手动创建）；
第3步：数据库表设计（先教用户表的设计思路，让我写建表SQL，你校验并纠错）；
第4步：后端基础框架搭建（教我用Hertz框架初始化后端项目，写main.go，逐行解释代码含义）；
第5步：第一个接口开发（用户注册接口，先讲逻辑，给核心代码片段，让我手动敲并调试）；
第6步：前端基础页面搭建（教我创建Vue3项目，写登录/注册页面的基础布局）；
第7步：前后端联调（教我配置前端请求地址，测试注册接口能否正常调用）；
第8步：核心功能拓展（视频上传、点赞等，按同样逻辑分步教）；
第9步：容器化配置（教我写Dockerfile，解释多阶段构建的每一行含义）。
```

### 步骤4：和导师的日常交互模板（你只需套话）

请帮我检查是否有问题，没问题的话教我下一步。

## 三、TikTok项目「分步教学」示例（第4步：后端基础框架搭建）
### 导师会这样引导你（参考，帮你理解预期）：
> 🌟 第4步：Hertz框架初始化后端项目（核心目标：让项目能启动，理解基础入口文件逻辑）
> 1. 先讲原理：Hertz是字节开源的Go后端框架，main.go是项目入口，核心是创建引擎、注册路由、启动服务；
> 2. 给核心示例（仅关键代码，不是完整代码）：
> ```go
> package main
> 
> import (
>     "github.com/cloudwego/hertz/pkg/app/server" // 导入Hertz核心包
> )
> 
> func main() {
>     h := server.Default() // 创建默认的Hertz引擎
>     h.GET("/ping", func(c context.Context, ctx *app.RequestContext) { // 注册测试路由
>         ctx.JSON(200, "pong")
>     })
>     h.Spin() // 启动服务
> }
> ```
> 3. 让你做：
> - 先在你的项目目录下创建`main.go`；
> - 手动敲上面的代码（注意：我故意留了一个小问题——没导入context包，你写的时候可能会发现，也可能没发现，我会帮你指出来）；
> 4. 你写完反馈后，导师纠错：
> > 你写的代码里少了`import "context"`，这个包是Hertz处理请求上下文必需的，补充后执行`go run main.go`，如果终端输出“server running on port 8888”，说明框架搭建成功了。
